"쿡쉐어는 1인 가구의 고충을 해결하고자 개발된 음식 공유 플랫폼입니다. 이 플랫폼은 1인 가구의 경제적 부담을 줄이고, 환경 보호를 목적으로 합니다. 특히, 적정량 이상의 음식 구매, 유통 기한 관리의 어려움, 증가하는 음식물 쓰레기 문제 등 1인 가구가 마주하는 다양한 문제점들을 지역 커뮤니티 내에서 음식을 공유함으로써 해결하고자 합니다."

Back-end : 데이터베이스 설계 및 관리와 페이지네이션 로직 구현
예약 도메인의 데이터베이스 설계를 담당, Spring Data JPA를 이용하여 데이터의 일관성과 효율성을 보장하는 CRUD 기능을 구현 데이터 처리의 성능 최적화를 위해 `Pageable` 인터페이스를 활용하여 복잡한 페이지네이션 로직을 개발, 이를 서버 측에서 구현하여 사용자의 데이터 요청 처리 속도를 개선

사용자 경험 개선을 위한
인터페이스 개발

사용자가 서비스를 쉽게 이해시키고 회원가입을 유도하는 온보딩 페이지를 일러스트 이미지와 Swiper 기능을 활용하여 동적으로 구현했습니다. 회원가입 시 사용자 입력의 디바운싱 처리를 통해 서버 요청을 최소화, Debounce 커스텀 훅을 적용하여 별도의 로직 작성 없이 효율적이고 일관된 방식으로 개발을 진행, 카카오 지도 API를 활용하여 사용자의 키워드 입력에 따라 반응하는 지도 검색 기능을 구현, 입력 상태가 디바운싱을 통해 최적화되도록 설계하여 사용자의 경험을 개선했습니다.

애플리케이션 상태 관리 및 라우팅 최적화
유지보수 간편하게 개발

react-router-dom을 사용하여 라우팅 로직을 중앙화, 간소화하여 동적인 페이지 구현을 최적화했습니다. 또한, Axios를 사용하여 서버 연동한 데이터를 Redux Toolkit의 createSlice 함수를 활용하연 게시판 기능의 상태 관리를 간소화시켜 유지보수성이 용이하도록 향상시켰습니다.
