신협리온브랜치
프로젝트 개요

신협모바일 웹 플랫폼 서비스 구축 프로젝트는 기존 서비스의 사용자 접근성과 편의성을 대폭 향상하기 위해 진행되었습니다. '어부바' 캐릭터를 활용해 브랜드 아이덴티티를 강화한 모바일 웹 사이트를 새롭게 구축하였으며, 웹 표준과 웹 접근성을 철저히 준수하여 웹 접근성 인증 마크를 획득했습니다. 이 프로젝트는 앱 설치나 공동인증서 없이도 사용자가 편리하게 접근할 수 있는 환경을 제공하는 것을 목표로 했습니다.

Shin Hyeop ReonBranch keyvisual

1. main page (메인 화면)

메인 화면에서는 animation과 Lottie를 활용해 '어부바' 신협 캐릭터를 추가하여, 브랜드 아이덴티티를 강화하고 사용자의 관심을 끌었습니다.

2. branch page (브랜치 화면)

브랜치 화면은 스크롤 시 헤더가 축소되며, 해당 지역의 협동 조합 이름이 노출됩니다. 또한, 상품 카드는 스크롤에 따라 자연스럽게 나타나도록 구현했습니다.

3. product page (상품 화면)

상품 화면에서도 스크롤 시 지역 협동조합의 이름과 마크가 노출됩니다. 사용자가 상품을 선택하면 해당 상품으로 이동하며, 스크롤할 때 화면에 나타나는 상품에 따라 탭이 자동으로 활성화 됩니다.

  • 1. main page (메인 화면)

  • 2. branch page (브랜치 화면)

  • 3. product page (상품 화면)

  • faq page

    아코디언 메뉴로 구성된 FAQ 페이지를 구현하여, 사용자가 질문과 답변을 쉽게 탐색할 수 있도록 했습니다.

  • event page

    animation과 CSS를 활용해 카드 뒤집기 효과를 적용한 이벤트 페이지를 구현했습니다.

  • navigation

    script를 사용하여 슬라이드 메뉴를 구현, 사용자가 메뉴를 쉽게 탐색할 수 있도록 했습니다.

개발하기 편리한 프레임워크 사용
컴포넌트화하여 작업

Vue 3와 Quasar 프레임워크를 활용해 재사용 가능한 컴포넌트를 설계하고 개발하여 페이지를 구성했습니다. 이러한 접근 방식을 통해 개발의 편의성을 높이고, 컴포넌트화된 구조로 효율적인 작업을 가능하게 했습니다.

신협 메인 컴포넌트 작업 구성 예시 이미지 - (오른쪽)각 컴포넌트로 (왼쪽)페이지를 구성

사용자에게 일관된 서비스를 제공하는
웹표준과 웹접근성 준수 및 테스트

웹 표준과 웹 접근성을 철저히 준수하고, QA 테스트와 다양한 디바이스에서의 디버깅을 통해 웹 접근성 인증 마크를 획득했습니다. 이를 통해 사용자에게 일관된 서비스를 제공하며 사용자 경험을 개선했습니다.

  • 기본탐색

    • 스크린 리더가 페이지의 내용을 논리적이고 일관된 순서로 읽는지 확인
    • 페이지의 헤딩과 ARIA랜드마크가 올바르게 구성되어 있는지 검사

  • 대체텍스트

    • 모든 이미지에 대체 텍스트가 제공되어 있는지 확인
    • 차트, 그래프 등 비시각적 콘텐츠에 대한 설명이 적절히 제공되어 있는지 검사

  • 폼 요소

    • 모든 입력 필드에 명확한 라벨이 있는지 확인
    • 오류 메세지가 명확하고 스크린 리더로 쉽게 인식될 수 있도록 구성되어 있는지 검사

  • 키보드 접근성

    • 모든 기능이 키보드만으로도 접근 가능한지 확인
    • 탭 순서가 논리적이고 직관적인 검사

  • 동적 콘텐츠

    • 동적으로 업데이트되는 콘텐츠가 스크린 리더에 의새 올바르게 읽히는지 확인
    • 드롭다운 메뉴, 모달 창 등의 상호작용 요소가 스크린리더로 접근 가능한지 검사

위의 방식대로 작업을 진행했습니다.
